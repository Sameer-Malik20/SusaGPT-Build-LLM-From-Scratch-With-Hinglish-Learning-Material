## 🏗️ The Pillars of AI Compliance

### 1. Data Minimization (GDPR Principle)
AI ko train karte waqt "Necessary" data hi use karo. Faltu user names ya addresses delete hone chahiye.
- **Right to be Forgotten:** Agar koi user bole "Mera data delete karo", toh kya aapka model use yaad rakhta hai? **Solution:** Model Unlearning or Retraining.

### 2. Explainability (The "Why")
Jab model bank loan reject karta hai, toh use batana padega ki "Kyu?" (Interpretability).
- **Tools:** SHAP, LIME (To explain black-box AI decisions).

### 3. High-Risk AI (EU AI Act)
Agar aapka AI recruitment, medical diagnostics, ya biometric identification kar raha hai, toh use "High-risk" maana jayega.
- **Rules:** Continuous human oversight and rigorous testing Mandatory hain.

## 🛡️ Governance Checklist for AI Startups

1. [ ] **PII Redaction:** Training data clean hai (No names/phones)?
2. [ ] **Terms of Service (ToS):** Kya user ko pata hai uska data "Model improvement" ke liye use hoga?
3. [ ] **Model Ancestry:** Kya aapka base model (Llama-3) commercial use ke liye allowed hai?
4. [ ] **Bias Audit:** Model kisi specific race ya gender ke khilaf biased toh nahi?

## 🧪 Quick Test (Legal Logic)

### Q1: Fine-tuning on Customer Data
"Hum apne saare customer emails se model train karenge bina unse puche." --- **Kya ye safe hai?**
**Answer:** Bilkul nahi (GDPR Violation). Aapko user consent aur "Opt-out" ka option zaroori chahiye.

### Q2: Copyrighted Data
"Internet se ebooks utha kar model train karna legal hai?"
**Answer:** Gray area (Fair Use). Par badhe platforms (e.g. OpenAI) par lawsuits chal rahe hain. Industry standard: "Licensed datasets" use karo.
